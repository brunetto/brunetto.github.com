What is Docker

Well, Docker is

“An open source project to pack, ship and run any application as a lightweight container.”

I have written a brief introduction here, and the Internet is full of definitions and suggestions. Check also out this for a comparison with LDX.

However, in short, Docker:

  • is an isolated sharable environment (dev and ship)

  • avoid the need to mess-up host OS to change dev env

  • avoid the need to get mad to install an app

  • is (was) based on lxc (linux containers) and Go

  • is NOT a virtual machine → lightweight

  • is a secure (sandbox)

  • allow you to commit changes

  • is based on images and containers: templates and running instances (that lose memory on shutdown)

How to install Docker

There are many different way to install Docker. On *buntu, I find this the best (see also here).

  1. Add docker repository key to apt-key for package verification and add repository:
sudo sh -c "wget -qO- https://get.docker.io/gpg | apt-key add -"
sudo sh -c "echo deb http://get.docker.io/ubuntu docker main\
> /etc/apt/sources.list.d/docker.list"
  1. Install docker
sudo aptitude update && sudo aptitude install lxc-docker
  1. Avoid typing sudo everytime
sudo groupadd docker
sudo gpasswd -a ${USER} docker
sudo service docker restart

then logout and login again.

If you want, you can allow memory limits: set /etc/default/grub key GRUB_CMDLINE_LINUX to

GRUB_CMDLINE_LINUX="cgroup_enable=memory swapaccount=1"

then update GRUB with sudo update-grub and reboot.

Usefull commands

  • docker attach, create, exec, inspect, logs, push, pull, rm, rmi, run, start, stop, tag

  • Inside a container, exit with ctrl+p ctrl+q

  • Attach to a running container with docker attach <container name>
